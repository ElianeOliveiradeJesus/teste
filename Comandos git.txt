O que é ramificação (Branch) ? 
  Pontos na história do seu projeto.

Master é a parte principal do projeto. Sendo a parte inicial e finalizada do projeto!
Quando é necessário fazer alterações porém não vai interferir na pasta principal do projeto é criado a branch. 

Branch 1 - Desenvolve a funcionalidade login 
Branch 2 - Desenvolve a funcionalidade cadastro 
Branch 3- Desenvolve o formulário 

Após as funcionaliades estarem  prontas para unificar o que já foi feito com o projeto na master . Qual o comando devemos usar no git ? 

git merge 


Commint: É uma postagem das alterações feitas no projeto . 

Comando : 
git commit -m "Descrição do que foi realizado"

Merge: É junção da branch que você desenvolveu com a  funcionalidade onde é necessário unificar com a master. 

Comando : 
git merge 

Obs: Se não foi alterado a mesma linha do código , não haverá problemas . 


Remote : Ele faz com que o repositório que está na minha máquina poderá ser atualizado no servidor . 

Comando : 
git remote 

Push :  Ele adiciona o commit que você realizou na máquina para o servidor 

git push 

Pull: Ele puxa o que está no servidor para sua máquina local . 

git pull



Principais comandos do git  
git init  : Inicializa o git 
git add nome do arquivo : Cria uma fila de espera para ser postados 

git add . : Cria uma fila de espera de todos os arquivos para ser postados 

git status: Mostra as mudanças para ser postadas 

git commit -m "Descrição do que foi realizado" : Ele faz um comentário do que você alterou no projeto.
